# ADR-002: 为什么选择 Higress 作为 AI 网关

## 状态

- **日期**: 2024-01-20
- **状态**: 已接受
- **作者**: AI Guru Team

## 背景

ResolveAgent 需要一个 AI 网关来处理：
1. LLM 流量管理（路由、限流、熔断）
2. 多模型支持（Qwen、Wenxin、Zhipu、OpenAI）
3. 统一认证和鉴权
4. 可观测性（指标、日志、追踪）

## 决策

选择 **Higress** 作为 AI 网关。

## 考量因素

### 候选方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| Higress | AI 场景优化、Wasm 扩展、阿里云生态 | 相对较新 |
| Kong | 成熟、插件丰富 | AI 特性不足 |
| Envoy | 云原生标准 | 配置复杂、AI 能力需自建 |
| 自建 | 完全可控 | 开发成本高 |

### Higress 优势

1. **AI 场景优化**
   - 内置 LLM 路由
   - Token 级限流
   - 模型降级/熔断

2. **Wasm 扩展**
   - 支持 Go/Rust/AssemblyScript
   - 自定义插件

3. **Envoy 基础**
   - 云原生标准
   - 高性能

4. **阿里云生态**
   - 通义千问集成
   - 云服务集成

## 集成模式

1. **Route Sync**: Go Registry → Higress
2. **LLM Proxy**: Python Runtime → Higress → LLM Providers
3. **统一认证**: JWT + API Key

## 结论

Higress 专为 AI 场景设计，与 ResolveAgent 的架构需求高度契合。
